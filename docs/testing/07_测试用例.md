本文定义 remoPhoto 核心功能的长期回归基线，供 Alpha 发布验证和后续开发使用。测试范围来自：

## 2. 测试原则

### 2.1 优先级

| 优先级 | 定义 | 发布要求 |
| --- | --- | --- |
| P0 | 安装、启动、数据安全和核心闭环；失败会阻断主要使用 | Alpha 发布前全部执行且通过 |
| P1 | 重要功能、异常恢复和兼容行为；失败有明显影响但存在规避方式 | 变更涉及模块时必须执行 |
| P2 | 边界、体验、性能基线或长期质量项 | 按风险和设备条件执行 |

### 2.2 测试层级

| 层级 | 适用范围 | 当前策略 |
| --- | --- | --- |
| JVM | 纯计算、排序、分页、查找、DTO/manifest 校验 | 优先自动化 |
| Android 集成 | Room 迁移、DataStore、SAF、WorkManager、缓存 | 使用 AndroidX Test |
| Compose UI | 导航、状态、设置、分页、空错页面 | 使用 Compose UI Test |
| HTTP 契约 | API 路由、字段、状态码、兼容性 | MockWebServer 或本地服务 |
| 真机 E2E | SAF、前台任务、mDNS、图片加载、系统生命周期 | Alpha 必须保留 |

### 2.3 结果定义

- `PASS`：全部断言满足，Crash buffer 无新增崩溃。
- `FAIL`：至少一个断言不满足。
- `BLOCKED`：测试环境或前置条件无法满足，必须记录原因。
- `N/A`：当前测试范围明确不适用，必须记录裁剪依据。

失败用例必须保留最小复现步骤、版本、环境、脱敏日志和关联缺陷 ID。

## 3. 测试环境

### 3.1 设备矩阵

最低建议：

| 角色 | 要求 | 用途 |
| --- | --- | --- |
| 设备 A | Android 10 / API 29 | minSdk、SAF、基础兼容 |
| 设备 B | Android 13 或 14 | 通知权限、后台和主流系统行为 |
| 设备 C | Android 15 / API 35 | targetSdk 35 行为 |
| 远程组合 | 至少两台真实 Android 设备，同一 Wi-Fi | mDNS、HTTP、离线恢复 |

设备不足时，P0 本地流程至少在一台真机执行；远程流程不得仅用模拟器结论替代真机结论。

### 3.2 标准数据集

| 数据集 | 内容 | 用途 |
| --- | --- | --- |
| D0 空状态 | 无仓库、无分类 | 首次启动和空页面 |
| D1 小仓库 | 根目录图片；两级子目录；PNG/JPG/JPEG/GIF/WebP/BMP；大写扩展名；中文、空格、Emoji 文件名；损坏图片；非图片文件 | 功能和格式兼容 |
| D2 分页边界 | 分别含 0、1、20、21、40、41 个相册 | 默认每页 20 的边界 |
| D3 重名与排序 | 大小写、数字、同名、同图片数、同修改时间相册 | 稳定排序和模糊定位 |
| D4 本地大仓库 | 数千相册、数万至数十万图片 | 扫描、内存和后台稳定性 |
| D5 远程大仓库 | 建议复用 5225 相册、267995 图片历史样本或等价生成数据 | 远程同步和 262 页分页 |
| D6 备份样本 | 当前有效包、旧格式包、损坏 ZIP、非法条目、过高版本 manifest、损坏数据库 | 导入校验和回滚 |

测试数据不得包含未经授权的私人图片。大数据集应记录生成方式、相册数、图片数和总容量。

## 5. 通用真机证据

```powershell
$serial = "<adb-serial>"
$apk = "app\build\outputs\apk\release\<signed-apk-name>.apk"

adb -s $serial install -r $apk
adb -s $serial logcat -c
adb -s $serial shell am force-stop com.remophoto
adb -s $serial shell am start -W -n com.remophoto/.MainActivity
adb -s $serial shell dumpsys package com.remophoto
adb -s $serial shell uiautomator dump /sdcard/remophoto-ui.xml
adb -s $serial logcat -b crash -d
adb -s $serial shell dumpsys meminfo com.remophoto
```

每个 P0 真机用例至少记录：

- 日期、Commit、versionName/versionCode、APK SHA-256。
- 设备型号、Android 版本、安装方式。
- 数据集规模和网络条件。
- 实际结果、截图/录屏、Crash buffer。
- 关键 AppLogger 摘要；日志必须脱敏。
