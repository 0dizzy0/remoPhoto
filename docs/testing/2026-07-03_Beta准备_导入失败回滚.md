## 故障注入

导入器增加仅供模块内测试调用的阶段观察器，生产入口使用无操作实现，不存在全局可变测试开关。测试按以下顺序执行：

1. 创建含“原分类”和“原设置”的当前数据库与设置文件。
2. 创建含“导入分类”和“导入设置”的合法 Backup-v1 ZIP。
3. 完成数据库替换和设置恢复。
4. 在 `SETTINGS_RESTORED` 阶段注入 `IOException`。
5. 由现有异常路径执行数据库与设置回滚。

## 环境说明

vivo 系统会对首次安装 64 MB Debug APK显示人工安全验证，导致 Gradle UTP 的流式安装超时。完成一次机主验证后，使用 `adb install --no-streaming` 安装测试 APK并直接运行 AndroidJUnitRunner。该问题发生在测试启动前，不属于应用或回滚逻辑失败。

测试结束后已卸载 `com.remophoto.debug` 和 `com.remophoto.debug.test`，设备仅保留公开版 `0.1.0-alpha.4`。

## 残留风险

本测试覆盖“设置已恢复后抛出应用级异常”的完整回滚路径，不模拟写文件中途断电、文件系统损坏或进程被系统直接杀死。此类故障注入只在出现真实缺陷或 Beta 风险评估要求提高时追加。
