# SMB M2 验证记录

日期：2026-07-12
分支：`codex/smb-support`

## 验证范围

本轮只验证 M2 的数据、安全与生命周期骨架。Android 10/API 29、Samba/NAS、10,000+ 条目压力、完整异常矩阵与发布级兼容门禁按已确认决定延后至 M5。

已验证内容：

- Room v4→v5：新增 `domain`、`root_path`、`identity_key` 与唯一索引，旧 HTTP 连接回填身份键。
- v4 原始数据库备份导入后由 Room 无损迁移至 v5，并将导入连接重置为离线。
- HTTP/SMB 身份规范化、同主机不同共享/根目录/账号去重语义、上级路径拒绝。
- Room 元数据与 Keystore 凭据的创建补偿、实际 Keystore 保存/读取/原子替换/删除、重新认证失效旧会话、缺凭据恢复和删除后外部清理状态。
- SMBJ `0.14.0` 正式依赖的 SMB2/3 dialect 白名单、Guest/匿名拒绝、并发限制、操作超时、协程取消、资源反向关闭、只读 access mask 与稳定错误分类。
- Release 日志只记录连接 ID、阶段、计数、耗时、dialect 与错误分类，不记录端点、共享、账号、路径、凭据或第三方异常原文。

真机：vivo V1981A，Android 12/API 31，arm64-v8a。adb 流式安装在该设备上长时间无响应，改用 `adb install --no-streaming` 后稳定完成；该现象只影响本地部署方式。

## 保留项

- M2 不提供最终 SMB 添加 UI、目录递归索引或图片读取，这些分别属于 M4 与 M3。
- 真实 Windows SMB 的认证和共享根枚举已在 M0 独立 Spike 上得到 `PASS: entries=63`；正式 app 内的端到端添加/扫描/浏览将在 M3～M4 链路接通后验证。
- API 29 与完整发布矩阵必须在 M5 上线前补齐。
