# 0.1.0-beta.1 双机远程回归记录

## 1. 结论

`0.1.0-beta.1` 双机远程发布门禁通过。两台真机在同一可信 Wi-Fi 下完成同签名覆盖升级、HTTP 前台服务、mDNS 自动发现、远程同步、263 页分页、缩略图与原图、缓存清理、服务离线和恢复、Release 日志隐私及 Crash/ANR 检查。

共享端实际提供 5249 个本地相册、268871 张图片。客户端同步后显示 263 页；服务停止时强制刷新进入“错误”状态但保留缓存索引，服务恢复后原连接重新变为“已连接”，未创建重复仓库。

## 3. 覆盖升级与服务

- 共享端从公开签名 `0.1.0-alpha.3` 覆盖安装到 `0.1.0-beta.1`，`firstInstallTime` 保持不变。
- 覆盖后冷启动 `262 ms`，本地仓库索引仍存在。
- Android 16 上远程服务以前台服务运行，持续通知存在。
- 停止服务后 HTTP、mDNS 和前台 Service 停止；恢复后重新进入运行状态。
- 全程 Crash buffer 为空。

## 4. 发现、连接与同步

- 客户端打开“添加远程仓库”后，mDNS 自动发现 1 台共享端，端口为 `8080`。
- 从发现结果连接成功，无需手动输入地址。
- 共享 API 返回 5249 个本地相册，`imageCount` 合计 268871。
- 共享端还缓存有其他远程仓库，但 API 合计与本地仓库图片数完全一致，未发生远程仓库二次转发。
- 客户端同步后显示 268871 张图片和 263 页相册。

## 5. 分页与图片

- 第 1 页、第 2 页和第 263 页均可进入，页面内容发生对应变化。
- 第 263 页可显示末页相册和封面，没有分页越界或空白页。
- 抽查 25 张和 27 张图片的远程相册，图片网格正常生成。
- 全屏原图可打开，横向滑动后从第一张切换到下一张。
- 抽查期间客户端峰值 `TOTAL PSS` 约 159 MB，无 OOM、ANR 或崩溃。

## 6. 缓存、离线与恢复

缓存验证：

1. 浏览相册和全屏图片后，设置页显示缩略图缓存 `1.4 MB`、原图缓存 `3.8 MB`。
2. 执行“清除远程缓存”后，两项均更新为 `0 B`。
3. 再次浏览远程图片后，缩略图缓存恢复为 `420 KB`、原图缓存恢复为 `4.1 MB`。

离线恢复验证：

1. 停止共享端远程服务。
2. 客户端强制刷新后，仓库状态由“已连接”变为“错误”。
3. 已缓存的相册索引和 263 页分页仍保留，失败刷新没有清空旧数据。
4. 恢复共享端服务并再次强制刷新，原仓库恢复为“已连接”。
5. 恢复后远程图片网格、全屏原图和滑动翻页继续可用，没有创建重复仓库。
