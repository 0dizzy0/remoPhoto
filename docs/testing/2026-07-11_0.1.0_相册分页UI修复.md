# 0.1.0 发布后相册分页 UI 修复验证

## 结论

相册列表分页的四项发布后 UI 反馈已在 `codex/album-pagination-smoke` 分支修复，并在 vivo V1981A / Android 12 的现有大仓库正式包上完成复核。翻页回顶、返回保留页码、分页条稳定和省略号自由跳页均通过。

本次属于低风险 UI 小修，没有修改数据库、备份格式、远程协议、签名配置或 targetSdk，不执行双机远程、备份恢复和全量发布回归。

## 修改范围

- 页码变化后，网格和列表的 Lazy state 均滚动到首项。
- 相册列表路由重新进入时，仅在仓库或分类目标真正变化时重新加载，避免把当前页重置为第一页。
- 总页数大于 7 时使用稳定的五槽分页布局；第一、二、三页保持 `1 2 3 … N`。
- 省略号使用至少 48dp 的可点击目标并打开跳页对话框；当前页码不再打开该对话框。
- 关键状态保留和翻页回顶增加 `AppLogger` 日志。

## 验收边界与修正

首次 smoke 只安装独立 Debug 包并执行分页状态断言，不能证明正式包大仓库的完整 UI 旅程。由于正式包与 Debug 包可并存，手工打开 `com.remophoto` 时仍会看到旧代码。后续 UI 缺陷验收必须先记录被测包名，并把规则测试、测试包验证和正式包真实页面验证分开报告；不得以分页模型测试代替真实导航与滚动验收。
