## 故障根因与修复

- 根因：完整 BFS 目录扫描被包在 `SmbSessionManager.execute()` 的单次 30 秒超时中，大共享必然触发整轮超时；WorkManager 重试只会重复从头扫描。
- 修复：目录快照扫描只限制建连时间，单次 SMB 请求继续由 SMBJ socket 超时保护；整轮扫描由 WorkManager 取消信号和资源上限控制。
- 性能：用户先选择相册根目录以缩小扫描树；大型目录不再二次复制排序；每 500 张图片或 10 个目录上报进度；spool 与 500 条数据库批次继续保证有界内存和原子索引切换。
- 诊断：传输层异常稳定映射为网络不可达，Worker 输出具体安全错误分类，UI 显示可操作文案而不是 `RemoteDataException` 类名。
- 相册语义：扫描起点不再生成可见“根目录”相册；直属子目录提升为顶层相册，扫描起点中的直属图片归入“未分类图片”。
- 状态恢复：父级“添加远程仓库”对话框可见状态改用 `rememberSaveable`，使内部协议和非敏感表单状态能跨配置变化与进程恢复；密码继续排除在 SavedState 之外。
- 测试一致性：设备端刷新测试不再假设根级图片移除后仍存在“根目录”相册，改为验证“未分类图片”按需删除、子目录相册 ID 稳定和远端图片删除生效。

## 维护者范围决策（2026-07-13）

- 本轮暂不调整或强制 SMB 消息签名策略；继续以用户主动配置的可信局域网为产品边界，签名策略不计入 M4 退出证据。
- SMBJ `0.14.0` 上游 `PacketReader` 断线线程泄漏问题暂不纳入当前门禁。等待上游补丁或新版本发布后，再执行依赖升级、Android/R8 兼容性和断线资源专项回归；当前业务层不增加侵入式兼容补丁。
- SMB 添加对话框保留现有实现，本轮不进行 ViewModel 状态机迁移、组件拆分或大范围 UI 优化；相关解耦与体验优化后续作为独立任务处理，不阻塞 M4 功能验收。
- 项目不引入 TalkBack SDK、依赖或专项适配层，TalkBack 人工听觉验收不作为 SMB M4 或当前项目门禁；保留按钮说明、密码语义、合理焦点顺序和 48dp 点击区域等基础可访问性规范，并继续使用 Compose 语义树/UIAutomator 自动检查明显回归。
- remoPhoto HTTP 与 SMB 不建立功能依赖。共享远程仓库入口、协议路由、刷新/取消状态和删除生命周期继续由自动化回归覆盖；需要第二台设备的 HTTP 发现与手动添加端到端测试移至后续综合回归/发布前执行，不阻塞 SMB M4。
