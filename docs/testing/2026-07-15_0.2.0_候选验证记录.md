## 6. Smoke 脚本加固

候选验证暴露并修复了三个测试基础设施问题：

1. WSL 保活命令经 `Start-Process` 参数拼接后提前退出，导致约 20 秒后 WSL 自动停止；改为直接执行 `wsl.exe ... -- sleep infinity`，启动后断言进程存活并记录 PID。
2. Release 构建关闭 Debug Logcat，不能等待 `AppLogger.d` 的资源关闭文本；增加显式 `ReleaseMode`，以全屏图片 UI 语义作为 Release 媒体证据，资源释放继续由 instrumentation 断言。
3. 网络恢复后 WorkManager 可能先于脚本自动重试，刷新按钮变成“取消刷新”；脚本现在复用正在执行的任务，避免重复刷新竞态。

这些失败均发生在测试基础设施或 Release 可观测方式，不是产品功能回归；最终候选使用修复后的脚本完整通过。
