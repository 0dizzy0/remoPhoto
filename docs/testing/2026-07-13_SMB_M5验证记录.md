- Debug 与 Release 均打开静态图；动态 GIF 使用两份有效 PNG 帧哈希不同作为辅助证据，主断言使用 SMB 流资源完成日志和 UI tree 状态。
- 最终 Release 覆盖安装后仓库保持 `30 张图片`、`已连接`，WorkManager 无运行或 pending 作业。

本轮发现并修复三个最低版本/局域网环境缺陷：

1. Android 10 SQLite `3.22` 不支持 `VACUUM INTO`。数据库导出改为按 SQLite 版本选择路径：新版本继续使用 `VACUUM INTO`，旧版本在关闭 WAL 的受控事务中复制数据库并执行 `PRAGMA integrity_check`；API 29 instrumentation 由 9/12 恢复为 12/12。
2. API 29 模拟器的局域网连接未被系统标记为已验证互联网，`NetworkType.CONNECTED` 使 SMB Work 永久不运行。SMB 是用户配置的 LAN 端点，改为由实际 SMB 连接和有界重试判断可达性；对应 Work request 单测固定为 `NOT_REQUIRED`。
3. Samba NT1-only 会在 SMB2 协商阶段关闭连接，SMBJ 0.14.0 最终表现为 EOF。仅在 `client.connect()` 协商阶段把该 EOF 映射为 `UNSUPPORTED_DIALECT`，后续媒体读写 EOF 仍按断线处理；真实 NT1-only 服务端复测通过。
