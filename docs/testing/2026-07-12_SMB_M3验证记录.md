# SMB M3 验证记录

日期：2026-07-12
分支：`codex/smb-support`

## 范围

本记录只覆盖当前 M3 功能开发，不包含已决定延后到功能完成后、上线前执行的 API 29、Samba/NAS、完整异常矩阵和发布级资源压力门禁。

## 已实现

- SMB 共享根和可选子目录校验；路径拒绝 `..`、NUL 和目录项分隔符，保留原始大小写并统一 Unicode NFC。
- 递归目录快照、扩展名图片过滤、根相册、父子相册关系、纯空目录隐藏、重解析点跳过。
- 最大深度 32、最大条目 250,000、最大 spool 256 MiB；关键阶段记录连接 ID、仓库 ID、批次、数量、耗时和错误类别，不记录端点、账号、文件名或路径。
- 图片元数据顺序写入磁盘 spool；Room 仅在完整枚举成功后事务切换，500 条一批；重复刷新复用相册 ID，远端删除只在成功刷新后生效。
- `ImageEntity.filePath` 使用 `smb-media://connectionId/opaqueKey/variant/version`，不包含 host、share、username、domain 或远端路径；相册定位元数据通过带完整性校验的内部编码恢复相对目录，无需 Room v6。
- Coil Fetcher 按连接 ID 反查 Room 和 Keystore，通过 SMBJ 只读 access mask 打开流；Coil 关闭 `ImageSource` 时反向关闭 input/file/share/session 并释放并发许可。
- 缩略图采用客户端采样解码，复用现有 `remote_thumb_cache`；原图/GIF 复用 `remote_image_cache`，请求显式设置确定性 memory/disk cache key。
- SMB 刷新 Worker 只在网络连接时运行；认证、权限、路径和协议错误不重试，网络不可达、超时及未知瞬时错误最多共 3 次指数退避。
- 删除仓库前等待取消对应 SMB Work；Room 删除成功后按连接的确定性键定向清理远程缩略图/原图缓存，再删除 Keystore 凭据并关闭 session。

## 未完成的 M3 实机证据

当前正式 app 尚无 SMB 配置表单，该入口属于 M4；因此无法在不新增临时敏感参数通道的前提下，把用户现有 Windows 共享及密码安全地注入正式 M3 链路。以下证据在 M4 接通配置入口后立即补齐：

- 用户现有 Windows 大共享的真实递归扫描、分页浏览、静态图和 GIF 打开。
- 真实网络断开/恢复、扫描取消和远端新增/修改/删除后的手动刷新。
- 若目标目录达到 10,000+ 图片，记录真实耗时、spool 大小、峰值资源和连续浏览结果；若不足，则使用同一局域网测试共享补足规模。

这不阻塞进入 M4 开发，但在上述证据完成前，不把 M3 的真实 10,000+ 仓库退出条件标记为最终闭环。
