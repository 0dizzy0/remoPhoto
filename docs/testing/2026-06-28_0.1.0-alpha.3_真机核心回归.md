## 2. 设备与数据

| 角色 | 设备型号 | Android | ABI | 网络 | 仓库规模 |

测试数据集：

- 本地小仓库：根目录和两个子目录各 1 张 PNG，共 3 张图片、3 个相册。
- 本地边界仓库：在小仓库基础上增加 21 个单图目录，共 24 张图片、24 个相册；每页数量为 20。
- 非图片输入：有效备份 ZIP 和伪造损坏 ZIP，重扫时未被计入图片。
- 备份样本：应用内导出的 Backup-v1 ZIP，约 3.15 KB。
- 全部测试图片由项目 QA 截图生成，不包含用户私人图片；测试后已删除应用仓库和手机测试目录。

未执行的模块与原因：

- 远程客户端包：只有一台设备，无法验证 mDNS 跨设备发现、添加远程仓库、远程缓存、离线隐藏和恢复。
- 覆盖升级：设备原有 Alpha 2 与当前 Release APK 签名不同，`adb install -r` 返回 `INSTALL_FAILED_UPDATE_INCOMPATIBLE`；用户卸载旧版后完成全新安装。
- 备份失败回滚：未注入数据库替换中途失败，`TC-BKP-010` 未执行。
- 全屏多图手势：测试相册每个目录只有 1 张图片，未覆盖左右翻页、缩放、音量键和自动播放。
- 迁移包：没有可用的同签名旧版安装包和历史数据库 fixture。
- 性能专项：本轮使用小型合成数据，不执行 D7 大仓库性能基线。
