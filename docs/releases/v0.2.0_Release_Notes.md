# remoPhoto 0.2.0 Release Notes（草案）

日期：2026-07-15

状态：已于 2026-07-15 正式发布。Tag `v0.2.0` 指向合并提交 `d68a89f`；正式签名、API 29 升级/SMB Smoke、主分支 CI 和线上附件复验均已通过。

## 新增

- 支持把可信局域网中的 Windows、Linux Samba 或 NAS SMB2/3 共享添加为只读图片仓库。
- 支持显示名、主机、端口、共享名、可选相册根目录、用户名、可选域和密码；连接测试成功后才保存。
- 支持中文、空格和嵌套目录，静态图片与动态 GIF 使用与现有远程仓库一致的浏览体验。
- 支持手动刷新、取消、重新认证、修改相册根目录和删除仓库；网络或服务端故障时保留上一份完整索引。

## 安全边界

- 仅支持 SMB 2.0.2 及以上版本，不支持 SMB1/NT1，也不提供降级选项。
- 首版不支持 Guest/匿名、Kerberos、DFS 或对服务端文件执行写入/删除。
- 密码只保存在 Android Keystore，不进入 Room、应用内备份、日志、UI tree、图片引用或缓存键。
- 连接元数据（主机、共享名、用户名和子目录）可进入应用内备份，应把备份文件视为敏感数据。
- 功能只面向用户主动配置的可信局域网，不应通过公网或公共 Wi-Fi 暴露 SMB 服务。

## 兼容性与验证

- Android：Pixel 3 / Android 10 / API 29 AVD，以及 API 31、API 36 真机矩阵。
- 服务端：Windows SMB3 与 Ubuntu 24.04 LTS / Samba 4.19.5。未宣称覆盖所有商业 NAS 的厂商扩展。
- 自动化：67/67 JVM、Lint、Android 测试 APK、R8/minified Release、API 29 instrumentation 12/12。
- 运行时：Debug 与 minified Release 均完成认证、30 张中文层级相册索引、静态图和动态 GIF。
- 异常：错误密码、离线、共享移除、权限拒绝、SMB1-only、超时、取消及恢复均有稳定分类，失败/取消保留旧索引。
- 单机 Smoke：临时 Samba 共享、安装、添加、30 张同步、媒体流、断线恢复、远端 30→31→30、删除仓库与清理最终输出 `PASS`。
- Release 隐私扫描中 password、username、endpoint、共享名、路径和 fixture 名命中 0；Crash buffer 与 last ANR 中包名命中 0。

## 供应链

- Bouncy Castle：约束到 `bcprov-jdk18on:1.84`，避免解析到受 CVE-2026-0636 影响的 `1.79`。
