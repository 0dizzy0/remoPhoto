# remoPhoto 0.1.0-beta.1 Release Notes

> 当前状态：候选验证中，尚未发布。完成主力真机、双机远程和线上附件复验前不创建 Tag。

## 本次重点

`0.1.0-beta.1` 冻结大功能，集中验证从公开 Alpha 升级、备份恢复、数据完整性、远程稳定性和 Release 日志隐私。

## 主要变化

- 版本递增为 `versionCode = 5`、`versionName = 0.1.0-beta.1`。
- 固定相册分页、稳定排序、Unicode/空白名称和模糊定位行为。
- 固定备份 manifest/version、条目白名单、重复条目、路径安全和损坏 ZIP 的拒绝规则。
- 增加 Room v3→v4 migration 和备份导入失败回滚的可重复 Android 集成测试。
- 保留最小 GitHub Actions CI，对 JVM 测试、Android Lint 和 Debug 构建执行持续验证。

## 发布前验证

- [x] CI、JVM 测试、Lint 和 signed Release 构建通过。
- [x] `0.1.0-alpha.4 → 0.1.0-beta.1` 同签名覆盖安装，数据库与设置保留。
- [x] 有效备份导出和恢复通过；非法输入拒绝与失败回滚证据有效。
- [x] 主力真机完成本地核心、冷启动和 Crash buffer 检查。
- [ ] 两台真机完成发现、连接、分页、图片、缓存、离线和恢复回归。
- [x] Release 日志不包含 DEBUG、完整路径、URI、IP、设备名或用户自定义名称。
- [ ] APK 版本、签名、SHA-256、Tag、Release Notes 和线上附件一致。

## 安装与升级

- 当前公开版本仍为 `0.1.0-alpha.4`；本候选完成门禁后才会提供下载。
- 计划支持从公开 `0.1.0-alpha.4` 直接覆盖升级，并在发布前保留真机数据验证证据。
- 更早的内部 Alpha 2 使用不同签名，不能直接覆盖；应先导出备份，再安装公开签名版本并导入。

## 已知限制

- 远程服务使用无认证明文 HTTP，只适用于可信局域网。
- targetSdk 为 35，Android 16/API 36 专项计划在正式版前完成。
- SMB、视频、多用户和完整 UI 自动化不属于本次 Beta 范围。
