# remoPhoto 0.1.0-alpha.4 Release Notes

> 当前状态：发布候选，尚未创建 Tag 或 GitHub Release。

## 本次重点

`0.1.0-alpha.4` 是质量基础版本，不增加大型功能。它为个人项目建立最小 CI，补齐相册核心规则、备份输入边界和最近数据库迁移路径的自动化证据。

## 主要变化

- 增加 GitHub Actions：JVM 测试、Android Lint 和 Debug 构建。
- JVM 测试从 10 条增加到 19 条。
- 相册排序增加确定性次级顺序，覆盖分页、Unicode/空白名称和模糊定位。
- 备份导入统一校验 manifest/version、ZIP 类型、条目白名单、重复条目和路径安全。
- 增加 Room v3→v4 真机 migration 测试。
- Debug 包改用 `com.remophoto.debug`，可与公开签名版本共存。

## 安装与升级

- 新安装：下载发布页中的 signed APK 后安装。
- 从公开 `0.1.0-alpha.3` 升级：直接覆盖安装，已验证同签名数据保留。
- 更早的内部 Alpha 2 使用不同签名，不能直接覆盖；应先导出备份，再卸载旧包、安装新包并导入。

安装前请核对发布页 SHA-256。正式发布时在此处补充最终 hash 和下载链接。

## 已知限制

- 远程服务仍使用无认证明文 HTTP，只适用于可信局域网。
- targetSdk 仍为 35，尚未完成 Android 16/API 36 专项。
- SMB、视频和多用户能力尚未实现。
- 远程协议、失败回滚和 UI 自动化仍主要依赖按需手工回归。
