# remoPhoto 0.1.0 Release Notes

> 发布日期：2026-07-08；渠道：[GitHub Release](https://github.com/0dizzy0/remoPhoto/releases/tag/v0.1.0)。双机远程 Smoke、备份导出日志隐私抽查、升级数据保留和线上附件 hash/签名/版本复验均已通过。

## 本次重点

`0.1.0` 是首个稳定版本候选，基于 `0.1.0-beta.1` 的范围冻结结果，只纳入正式版前的兼容性修复、发布证据自动化和文档收敛。

## 主要变化

- 版本递增为 `versionCode = 6`、`versionName = 0.1.0`。
- 修复 Android 10（API 29）DownloadsProvider 返回 `raw:` document ID 时，仓库名和相册名显示完整内部路径的问题。
- 增加发布候选 APK 一键 Smoke 脚本，用于自动收集签名、安装、版本、冷启动、UI dump、截图、Crash buffer、进程日志、应用文件日志、基础 Release 日志隐私扫描和内存快照证据。

## 安装与升级

- 公开安装包只通过 GitHub Releases 提供。
- 已验证公开 `0.1.0-alpha.4 -> 0.1.0-beta.1` 和 `0.1.0-beta.1 -> 0.1.0` 覆盖升级。
- 升级前建议使用应用内“导出数据库”创建备份。
- 更早的内部 Alpha 2 使用不同签名，不能直接覆盖；应先导出备份，再安装公开签名版本并导入。
- 下载后可在 Windows PowerShell 中执行：

```powershell
Get-FileHash .\remoPhoto-0.1.0.apk -Algorithm SHA256
Get-Content .\remoPhoto-0.1.0.apk.sha256
```

两个值必须完全一致。首次安装需要允许当前文件管理器或浏览器“安装未知应用”，安装完成后应关闭该临时授权。不要从第三方 APK 站点下载。

## 已知限制

- 远程服务使用无认证明文 HTTP，只适用于可信局域网。
- targetSdk 为 35；targetSdk 36 升级独立安排。
- SMB、视频、多用户、HTTPS/TLS、访问令牌和完整 UI 自动化不属于 `0.1.0` 范围。
