## 重要已修复问题

### REL-007 Release 文件日志仍记录图片文件名

- 现象：2026-06-28 在 signed Release APK 真机回归中，623 行 AppLogger 日志没有 DEBUG、完整路径、URI、IP、设备别名或仓库名，但有 31 个 `fileName=<原值>`。
- 影响：图片文件名可能包含人物、地点或事件信息，用户分享诊断日志时会泄漏私人元数据；当前行为不满足 Release 日志统一脱敏声明和 `TC-OBS-002`。
- 根因：`ReleaseLogSanitizer` 没有覆盖 `fileName`，并且空封面告警使用的 `album="..."` 也不属于已有名称键。
- 修复：为带空格或引号的 `fileName` 增加整段脱敏；将 `album`、`repository`、`repo` 和 `device` 实体键纳入统一名称脱敏；保留计数和 ID。
- 验证：新增 2 个 JVM 测试，完整测试 8/8 通过。最终 signed Release APK 真机日志中，原始测试文件名、仓库名、未脱敏 `fileName` 和 `album` 均为 0；观察到 `fileName=<redacted>` 2 条、`album=<redacted>` 1 条。
- 残留风险：新增用户可控日志字段时仍需使用已有受保护键或同步扩充脱敏测试。

### REL-008 远程服务停止后 Service 记录残留

- 现象：关闭“允许远程访问”后，HTTP 已不可访问且前台通知移除，但 `dumpsys activity services com.remophoto` 仍显示 `HttpServerForegroundService` 为 started Service。
- 影响：没有发现残留网络访问，但 Service 和已注册的网络/亮屏回调会继续占用进程资源，生命周期状态不够干净。
- 根因：`ACTION_STOP` 只调用 `stopServer()`；该方法停止 HTTP/mDNS、释放锁并调用 `stopForeground()`，但没有调用 `unregisterRecoveryCallbacks()` 和 `stopSelf()`。
- 修复：停止命令改为 `START_NOT_STICKY`，注销网络/亮屏自愈回调并调用 `stopSelf(startId)`。
- 验证：最终 signed Release APK 启动服务后 `/api` 返回 200；关闭后约 3.5 秒内 `dumpsys` 不再显示 Service，API 不可访问；重新开启和第二次关闭均通过。
- 残留风险：不同厂商的 Service 回收时序可能不同，后续设备回归继续检查停止结果。

### REL-002 Release 产物未签名

- 现象：Release 只能生成 `app-release-unsigned.apk`，无法作为候选安装包发布。
- 根因：项目没有本地 signingConfig、密钥初始化流程或 CI 环境变量入口。
- 修复：增加 `.signing/signing.properties` 和 `REMOPHOTO_RELEASE_*` 双入口，提供不会覆盖已有密钥的初始化脚本；密钥和密码均由 Git 忽略。
- 验证：2026-06-27 生成 `app-release.apk`，`apksigner verify --verbose --print-certs` 显示 `Verifies`，APK Signature Scheme v2 验证通过。
- 残留风险：密钥尚需由项目所有者离线备份；APK 全新安装已通过，覆盖升级仍缺少同签名旧版基线。

### REL-005 Release 日志偏详细

- 现象：Release 默认记录 DEBUG、输出 Logcat，部分消息包含完整路径、URI、设备名和局域网地址。
- 根因：`AppLogger` 没有区分构建类型，也没有统一的敏感字段处理层。
- 修复：Release 最低等级改为 INFO、关闭 Logcat、过滤 DEBUG，并对 URI、路径、IPv4/IPv6 和用户自定义名称统一脱敏；直接输出完整 SAF URI 的路径已收敛。
- 验证：脱敏测试扩充到 5 个；完整 JVM 测试 8/8 通过，Lint 为 `No issues found`；Android 12 真机正常浏览日志抽查通过。
- 残留风险：后续新增日志字段时继续执行 Release 真机隐私抽查。

### REL-006 明文 HTTP 发布边界不清晰

- 现象：`usesCleartextTraffic=true` 允许明文流量，但发布边界和系统备份策略未最终确定。
- 根因：远程仓库需要连接动态局域网 IP，无法通过静态域名白名单表达。
- 修复：保留明文 HTTP 并在 README、安全说明和发布计划中限定为可信局域网；关闭系统备份，Android 12+ 云备份和设备迁移显式排除全部应用数据。
- 验证：合并后的 Release Manifest 为 `allowBackup=false`、`usesCleartextTraffic=true`，Lint 为 `No issues found`。
- 残留风险：当前没有 TLS 或访问令牌，不能用于公共 Wi-Fi 或公网。

### REL-003 Room schema 未导出

- 现象：`exportSchema = true`，但 KSP 未配置 schema 输出目录，构建持续提示缺少 `schemaLocation`。
- 根因：`app/build.gradle.kts` 没有向 KSP 传入 Room schema 输出路径。
- 修复：配置 `room.schemaLocation` 为 `app/schemas`，生成并纳入 v4 schema JSON。
- 验证：2026-06-27 执行 `:app:kspDebugKotlin` 成功，生成 `app/schemas/com.remophoto.data.local.AppDatabase/4.json`。
- 残留风险：当前仍缺少基于历史 schema 的自动 migration 测试。

### REL-001 Lint 质量门禁未通过

- 现象：历史记录为 1 个 Compose error、27 个 warning，Alpha 质量门禁未通过。
- 根因：历史 Compose error 在本轮复核前已不再复现；剩余问题包括 Compose 参数规范和高频状态读取、硬件 ID、无限 WakeLock、冗余 SDK 判断、无用资源和图标配置。
- 修复：逐项修正 25 条实际问题；对 API 36 独立升级和 AAPT 必需的 `mipmap-anydpi-v26` 目录保留 2 项局部、带原因的 `app/lint.xml` 例外。
- 验证：2026-06-27 执行 `:app:lintDebug`，结果为 `BUILD SUCCESSFUL` 和 `No issues found`；3/3 JVM 测试及 Debug APK 构建通过。
- 残留风险：targetSdk 35 升级到 36 需要单独兼容验证；Kotlin 编译器仍报告部分 Android API 弃用提示，不属于本次 Android Lint 告警。

### P4-F01 数据库迁移 FK 导致启动闪退

- 现象：旧版本数据库升级后启动闪退。
- 根因：迁移 SQL 中的 FK 定义与 Room Entity 期望不一致。
- 修复：移除 SQL 层 FK，由应用层维护远程连接关联。
- 验证：覆盖安装后可正常启动。
- 残留风险：后续迁移必须增加 migration 测试。

### P4-F02 ForegroundService 启动超时

- 现象：远程服务启动时设置页闪退。
- 根因：前台服务未在系统要求时间内调用 `startForeground()`。
- 修复：服务启动后立即进入前台通知，再异步启动 HTTP/mDNS。
- 验证：真机启动远程服务无新增 crash。

### P4-F05 mDNS 服务未注册

- 现象：同局域网设备无法发现当前设备。
- 根因：HTTP 服务启动链路没有实际调用 mDNS 注册，并缺少 multicast lock。
- 修复：在远程服务中接入 `MdnsRegistrar` 和 `WifiLockManager`。
- 验证：真机可注册和主动发现局域网服务。

### P4-F07 远程同步后相册不显示

- 现象：远程同步成功但列表为空。
- 根因：`loadAlbums()` 与同步协程互相取消，导致 UI 状态没有刷新。
- 修复：调整同步和加载顺序，远程仓库进入时优先展示缓存，再后台刷新。
- 验证：远程仓库列表可显示并分页。

### P4-F08 远程图片元数据未同步

- 现象：远程相册可见但进入后图片缺失。
- 根因：相册同步后未按需调用图片同步。
- 修复：Gallery 进入远程相册时触发图片元数据同步。
- 验证：远程相册图片可加载。

### P4-F09 默认 ImageLoader 不支持远程图片

- 现象：远程图片 URL 无法通过默认加载器显示。
- 根因：Coil 默认 ImageLoader 未注册 `RemoteImageFetcher`。
- 修复：全局 ImageLoader 和远程 thumbnail/image loader 均注册远程 fetcher。
- 验证：远程封面和图片可显示。

### P4-F10 mDNS 发现和注册不稳定

- 现象：发现失败、IP 获取异常或发现自身。
- 根因：权限、LAN IP 获取和实例过滤策略不完整。
- 修复：增加 `CHANGE_WIFI_MULTICAST_STATE`、多路径获取 LAN IP、稳定 instanceId 和自身过滤。
- 验证：两台真机局域网互相发现，且自身被过滤。

### P4-F15 数据库导入导出一致性不足

- 现象：导出活跃 DB/WAL/SHM 可能不一致，远程连接元信息兼容性不足。
- 根因：直接复制数据库文件，缺少 manifest、版本校验和回滚策略。
- 修复：改为 SQLite `VACUUM INTO` 快照，导出 manifest；导入前白名单解压、`integrity_check`、版本校验，替换前创建回滚快照。
- 验证：新旧格式导入路径可用，失败时可恢复旧数据。

### P4-F16 大仓库扫描内存与耗时问题

- 现象：数十万图片仓库扫描慢，内存压力高。
- 根因：扫描期间大量 `ImageFileInfo` 常驻内存，并且部分查询逐图片执行。
- 修复：SAF 单游标目录查询、磁盘 spool 流式暂存、批量写入、事务索引切换、前台 WorkManager。
- 验证：真机可启动前台扫描，进程保持存活。
- 残留风险：进程被系统杀死后仍可能重新遍历，目录队列级检查点未实现。

### P4-F17 大型远程仓库分页不显示

- 现象：远程仓库有 5225 个相册，但分页栏长期按 1 页显示。
- 根因：总页数不是响应式状态，UI 未订阅真实总页数。
- 修复：总页数改为 `StateFlow`，缓存优先展示，后台刷新。
- 验证：真机显示 `1 2 ... 262`，可跳到第 262 页。

### P4-F18 远程分类跨同步丢失

- 现象：远程相册重新同步后分类关联可能丢失。
- 根因：同步时删除并重建相册导致本地主键变化。
- 修复：按稳定远程路径复用本地相册主键，仅删除服务端已移除的相册。
- 验证：临时远程分类重同步后关联仍存在。
