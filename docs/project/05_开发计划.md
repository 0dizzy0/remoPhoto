## Beta 候选范围

Beta 阶段应解决 alpha 暴露出的稳定性问题，并验证公开升级路径。以下是候选能力，不是 `beta.1` 或 `0.1.0` 的默认承诺：

- SMB 电脑共享仓库。
- HTTPS/TLS 或局域网访问令牌。
- 扫描任务目录队列级持久化检查点。
- 更完整的 UI 自动化测试。
- 远程服务安全和隐私说明。
- README 的真实截图、安装包链接和更完整安装说明。

以下事项不建议阻塞 `0.1.0` 正式版：

- 视频播放。
- 多用户权限管理。
- Play Store 上架材料。
- 固定性能指标承诺。

## 技术债清单

| 项目 | 当前影响 | 建议处理时间 |
| --- | --- | --- |
| 自动化测试覆盖低 | 回归主要依赖手工 | 先覆盖高风险纯逻辑，UI 与专项测试按收益增加 |
| 明文 HTTP 无认证加密 | 可信局域网外不安全 | Alpha 保留边界说明，Beta 评估认证与 TLS |
| WorkManager 扫描恢复不精确 | 进程被杀后可能重新遍历 | `0.1.0` 后按真实反馈评估 |
| 文档历史堆叠 | 维护成本高 | 本轮已收敛 |

## 发布命令

基础门禁：

```powershell
.\gradlew.bat :app:testDebugUnitTest :app:lintDebug :app:assembleRelease --console=plain
```

发布前建议：

```powershell
git status --short
Get-FileHash app\build\outputs\apk\release\<signed-apk-name>.apk -Algorithm SHA256
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\run-smoke.ps1 `
  -BuildRelease `
  -ExpectedVersionName "0.1.0" `
  -ExpectedVersionCode 6
```

## 维护规则

1. 新任务进入本文件时必须带优先级和验收方式。
2. 已完成的大段过程不继续保留在本文件，只在[项目状态](项目状态.md)记录摘要。
3. 问题根因和修复细节写入[问题日志](06_问题日志.md)。
4. 测试设计写入[核心功能测试用例](../testing/07_测试用例.md)，单次结果使用[测试执行记录模板](../testing/测试执行记录模板.md)。
5. Alpha 历史门禁写入[首个 Alpha Release 计划](../releases/08_首个AlphaRelease计划.md)，Alpha 后、Beta 和正式版门禁写入[Beta 与正式版 Release 计划](../releases/09_Beta与正式版Release计划.md)。
