## 当前能力

### 本地仓库

- 添加、删除和重新扫描本地图片仓库。
- 使用 SAF 授权访问 Android 10+ 文件目录。
- 支持 PNG、JPG/JPEG、GIF、WebP、BMP。
- 子目录自动映射为层级相册。
- 大仓库扫描已引入 WorkManager、磁盘 spool、事务切换和阶段化进度。

### 相册与图片浏览

- 相册列表支持网格/列表展示、分页、相册排序和名称模糊定位。
- 相册内图片支持网格/列表展示和排序。
- 全屏浏览支持滑动、点击区域、音量键、鼠标滚轮、缩放和自动播放。
- 分类支持创建、编辑、删除和批量关联相册。

### 远程仓库

- 本设备可通过局域网 HTTP 服务共享本地相册。
- 支持 mDNS 自动发现和手动 IP 添加。
- 远程相册元数据可同步到本地 Room 缓存。
- 远程相册支持分页、封面、分类关联和离线状态处理。
- 远程仓库仅面向可信局域网，不承诺公网访问安全。

### 设置与维护

- 支持主题、深色背景类型、高对比度、排序、分页数量、播放间隔等设置。
- 支持数据库导入导出；远程连接元信息可导出，凭据不导出。
- 远程缩略图和原图缓存可统计和清理。
- 关键路径使用 `AppLogger` 记录可观测日志。

- Android 10/API 29 的 SAF DownloadsProvider 授权、扫描与本地图片浏览可用。
- 两台 Android 设备在同一局域网中可通过 HTTP/mDNS 进行远程仓库验证。
- 大型远程仓库样本 5249 个相册、263 页，分页和缓存优化后可用。
- 远程断网后分类页隐藏离线远程相册，网络恢复后可重新连接。

这些结果来自手工验证，不等同于完整自动化回归。

## 主要风险

| 风险 | 影响 | 当前处理 |
| --- | --- | --- |
| Release 密钥丢失 | 后续版本无法覆盖升级 | `.signing/` 必须离线加密备份 |
| 自动化测试仍不完整 | 远程协议和 UI 仍有部分真机依赖 | 已覆盖 67 条 JVM、v3→v4/v4→v5 migration、备份导入与 SMB instrumentation；其余按变更风险增加 |
| API 36 尚未升级 | targetSdk 仍为 35；API 29 与 Android 16 已完成基础兼容验证 | 正式版沿用已验证 targetSdk 35，targetSdk 36 升级独立安排 |
| 明文 HTTP | 局域网之外不安全 | 仅定位可信局域网 Beta 功能 |
| 后台与厂商限制 | 大仓库扫描/远程服务可能被系统限制 | 使用前台任务和日志定位，持续收集反馈 |

## 当前发布建议

当前公开版本为 `0.2.0`，对应 `versionCode = 7` 和 Git tag `v0.2.0`，已完成正式发布与线上附件复验。

Beta 发布定位：

- 面向作者和早期用户试用。
- 允许存在已知限制。
- 不承诺 Play Store 上架质量。
- `0.2.0` 新增可信局域网 SMB2/3 只读客户端；仍不承诺 SMB1、Guest、Kerberos、DFS、远端写入、视频、多用户、HTTPS 或公网访问。

下一目标是收集 `0.2.0` 实际使用反馈，并独立规划 targetSdk 36、SMB 兼容矩阵扩展和后续版本范围；若出现 P0/P1 或高风险兼容问题，优先准备修复版本。
