# remoPhoto 需求分析

更新时间：2026-06-23

## 1. 项目定位

remoPhoto 是一个 Android 本地/局域网相册管理应用。它面向以文件目录组织图片的用户，把用户授权的目录映射为相册，并提供大仓库场景下可用的浏览、排序、分类、全屏查看和局域网远程访问能力。

首个 alpha 的定位是个人开源项目的可试用版本，不承诺商业级稳定性或应用商店上架质量。

## 2. 目标用户

- 在 Android 手机上保存大量图片，并希望按目录结构浏览的用户。
- 使用 NAS、电脑、备用手机或平板保存图片，希望在局域网内临时浏览的用户。
- 关注本地文件结构和可控数据管理，不希望完全依赖系统相册聚合逻辑的用户。
- 愿意接受 alpha 版本限制并反馈问题的早期用户。

## 3. 核心使用场景

### S1. 本地大仓库浏览

用户选择一个或多个本地目录作为图片仓库。应用扫描目录、建立索引、按子目录生成相册，用户可分页浏览、排序、搜索定位相册，并进入相册查看图片。

### S2. 全屏快速阅图

用户进入相册后全屏浏览图片，使用滑动、点击区域、音量键或蓝牙鼠标滚轮快速切换图片；需要时可以缩放、自动播放或查看进度。

### S3. 分类整理

用户可以创建分类，把本地或当前已连接的远程相册加入分类。分类用于快速回到常用相册集合，不改变原始目录结构。

### S4. 局域网远程仓库

两台 Android 设备在同一局域网内，一台开启远程服务共享本地相册，另一台通过 mDNS 或手动 IP 添加远程仓库并浏览。远程仓库缓存到本地数据库，离线时按连接状态处理。

### S5. 备份与恢复

用户可导出应用数据库和设置，用于备份或迁移。远程连接元信息可以导出，但密码或凭据不导出。

## 4. Alpha 范围

首个 alpha 应覆盖：

- 本地仓库添加、扫描、删除和重新扫描。
- SAF 持久化目录授权。
- 图片格式：PNG、JPG/JPEG、GIF、WebP、BMP。
- 子目录映射为层级相册。
- 相册列表网格/列表、分页、排序、名称模糊定位。
- 相册内图片浏览、排序、缩略图加载。
- 全屏浏览、缩放、音量键、鼠标滚轮、自动播放。
- 分类管理和相册关联。
- 设置页基础配置。
- HTTP/mDNS 远程仓库。
- 远程相册元数据缓存、封面和离线行为。
- 数据库导入导出。
- 关键路径日志。

## 5. 非目标

首个 alpha 不承诺：

- PC SMB 仓库。
- 视频播放。
- 多用户权限管理。
- 公网访问。
- HTTPS/TLS 加密。
- 云同步。
- 完整 UI 自动化测试覆盖。
- 所有 Android 厂商后台策略兼容。
- Play Store 上架材料。

## 6. 功能优先级

| 优先级 | 范围 | 发布要求 |
| --- | --- | --- |
| P0 | 本地仓库、扫描、相册、图片浏览、基本设置 | alpha 必须可用 |
| P0 | release 构建、签名、smoke test、崩溃排查 | alpha 必须完成 |
| P1 | 远程仓库 HTTP/mDNS、远程缓存、分类离线行为 | alpha 纳入，但允许列已知限制 |
| P1 | 导入导出、日志、Room schema | alpha 前尽量完成 |
| P2 | 自动化测试、日志脱敏、安全边界优化 | alpha 后第一轮 |
| P3 | SMB、视频、多用户、HTTPS | beta 或更后续阶段 |

## 7. 质量要求

### 构建要求

- Debug 构建通过。
- Release 构建通过。
- Release 包可签名并安装。
- 发布前 `lintDebug` 至少无 error。

### 稳定性要求

- 冷启动无崩溃。
- 覆盖安装无数据库迁移崩溃。
- 本地仓库扫描过程中应用不应 ANR。
- 远程仓库连接失败时应保留已缓存数据并给出明确状态。

### 性能要求

Alpha 不承诺固定性能指标，但必须满足：

- 大仓库进入相册列表时 UI 可操作。
- 扫描、同步、导入导出等长任务有阶段性进度或日志。
- 大型远程仓库分页不应阻塞主线程。

### 可观测性要求

关键流程必须保留可定位日志：

- 本地仓库扫描开始、阶段进度、完成、取消和失败。
- 远程服务启动、停止、自愈、mDNS 注册和发现。
- 远程仓库同步开始、命中缓存、刷新、失败。
- 数据库导入导出开始、校验、替换、回滚和失败。
- 大仓库性能相关耗时摘要。

Release 版本日志应减少 DEBUG 和隐私路径输出。

## 8. 安全与隐私边界

- 本地图片路径、相册名和仓库名属于敏感信息，日志和导出文件应谨慎处理。
- HTTP 远程仓库仅用于可信局域网，不适合作为公网服务。
- 远程连接凭据不随数据库导出。
- 后续如果引入 SMB、HTTPS 或访问令牌，需要重新更新安全设计。
