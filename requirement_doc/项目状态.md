## Phase 4 范围确认（2026-06-12）

> 经用户确认，Phase 4 聚焦**远程仓库**功能。视频播放和多用户权限管理移入后续迭代。

### Phase 4 关键决策

| # | 决策事项 | 决定 |
|---|---------|------|
| 1 | 视频播放 + 多用户 | 移入后续迭代（Phase 5+） |
| 2 | 远程连接数据模型 | **方案 B**：新建 `remote_connections` 表，`RepositoryEntity` 通过 `remote_connection_id` FK 关联 |
| 3 | 凭据存储 | Android Keystore（AES/GCM），不入 DB、不随导出 |
| 4 | 导入导出范围 | Room DB + DataStore Preferences + remote_connections（清除凭据），不含缩略图 |
| 5 | 凭据导出策略 | **方案三**：导出保留连接元信息（IP/端口/设备名/用户名），凭据清除，导入后用户重输密码 |
| 6 | 离线处理 | 仓库级连接状态指示 + 相册级数据可用性标记，远程仓库独立 UI 分区 |
| 7 | 缓存策略 | L2 缩略图缓存与 HTTP/mDNS 同步开发（不延后） |
| 8 | 开发优先级 | **HTTP Server + mDNS 优先**，SMB 次之 |

### Phase 4 扫描启动闪退修复（2026-06-22）
