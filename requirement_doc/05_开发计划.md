## 六、Phase 4：远程仓库 ✅ 已完成（2026-06-12）

> **实际工期：** 1 天（Phase 4.1~4.7），SMB（4.8）后延
> **⚠️ 视频播放（原 P4-17~22）和多用户权限管理（原 P4-23~29）移入后续迭代（Phase 5+）。**

### 6.0 Phase 4 关键架构决策

| 决策 | 方案 | 说明 |
|------|------|------|
| 远程连接数据模型 | **独立 `remote_connections` 表** | `RepositoryEntity.remote_connection_id` FK 关联；null=本地 SAF 仓库 |
| 凭据存储 | **Android Keystore** | AES/GCM 加密，不入 DB，不随导出，key alias = `remote_conn_{id}` |
| 导入导出凭据策略 | **方案三：保留元信息、清除凭据** | 导出 IP/端口/设备名/用户名，导入后需手动重输密码 |
| 离线状态粒度 | **仓库级 + 相册级** | 仓库级连接状态指示（绿/灰/红），相册级数据可用性标记 |
| 远程 UI 组织 | **独立分区** | AlbumListScreen "本地仓库" / "远程仓库" 两个 Section |
| 缓存时机 | **与 HTTP/mDNS 同步开发** | L2 缩略图缓存不延后 |
| 开发优先级 | **HTTP/mDNS 优先，SMB 后置** | 先设备间互访，后电脑共享 |

## 九、风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| SAF 权限在部分 ROM 上兼容性问题 | 中 | 高 | Phase 0 提前适配主流厂商 ROM（MIUI、ColorOS 等） |
| 万级图片扫描性能不达标 | 中 | 高 | 分批扫描 + 索引优化 + 后台 Service |
| Coil GIF 动图内存/性能问题 | 中 | 中 | 限制同时播放 GIF 数量，全屏不可见时暂停 |
| mDNS 在部分路由器/ROM 不可用 | 高 | 中 | 提供手动输入 IP 的降级方案（P4.5-01）；测试主流路由器品牌 |
| JmDNS Android 兼容问题 | 中 | 高 | 手动指定 WLAN 网络接口绑定；必要时回退到 Android NsdManager |
| NanoHTTPd 后台被杀 | 中 | 中 | 前台 Service 保活，通知栏显示"远程服务运行中"，START_STICKY |
| 远程图片 ID 与本地 ID 冲突 | 低 | 高 | repositoryId 全局唯一区分；remote filePath 以 `http://` 或 `smb://` 开头 |
| 大尺寸原图 OOM | 中 | 高 | Coil 降采样 + Bitmap 复用池 + 内存 LRU；全屏加载 size(2048) 限制 |
| DataStore 导出/导入兼容性 | 低 | 中 | 导出前验证 Preferences 文件存在性；导入后逐 key 验证回写 |
| Keystore 跨设备不可恢复 | 高（预期行为） | 低 | 导入后 UI 引导用户重输凭据，不尝试恢复 Keystore |
| 缓存文件过大占用存储 | 中 | 低 | 缓存管理 UI 显示大小 + 手动清除按钮；LRU 自动淘汰 |

### 真机验证摘要

- 设备：V2254A，targetSdk 35，Debug 构建。
- 远程服务：HTTP、mDNS、三类锁及前台通知启动成功；切后台 15 秒后 `/api` 仍可访问。
- 封面：5224 个本地相册中 4425 个返回 `coverImageId`；抽样缩略图返回 JPEG 109298 bytes。
- 扫描闪退：定位为 Android 15 `InvalidForegroundServiceTypeException`，修复后 WorkManager FGS 类型为 `dataSync (0x1)`；持续观察 20 秒无新增 crash。
- 扫描内存基线：Java Heap 约 89.7MB、TOTAL PSS 约 262MB、TOTAL RSS 约 404MB；仅为单次 Debug 快照，不代表峰值。
