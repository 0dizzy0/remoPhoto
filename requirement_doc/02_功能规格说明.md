## 二、功能详细规格

### 模块A：图片仓库管理

#### A1 — 根仓库设置

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 用户选择本地文件夹作为图片根仓库，支持多个根仓库并存 |
| 输入 | 用户通过系统文件选择器或路径输入选择一个或多个文件夹 |
| 处理 | 保存路径列表到本地配置；验证路径可访问性 |
| 输出 | 根仓库列表，每个仓库显示路径、图片数量、最后扫描时间 |
| 约束 | 需适配 Android 存储权限（API 29+ Scoped Storage / SAF） |

**交互流程：**
1. 用户进入"仓库管理"页面
2. 点击"添加仓库"按钮
3. 系统弹出文件选择器，用户选择文件夹并授权
4. 系统扫描目录，统计图片数量
5. 仓库出现在列表中
6. 用户可长按仓库进行删除或重新扫描

#### A2 — 图片扫描与索引

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 递归扫描根仓库下所有图片文件，建立索引数据库 |
| 输入 | 根仓库路径列表 |
| 处理 | 递归遍历目录 → 过滤图片格式 → 提取元数据 → 写入索引数据库 |
| 输出 | 图片索引表（路径、文件名、大小、修改时间、所属相册等） |
| 触发 | 应用启动时自动扫描增量；用户手动触发全量扫描 |

**支持的图片格式（已确认）：**
- `.png` — PNG 静态图片
- `.jpg` / `.jpeg` — JPEG 图片
- `.gif` — GIF 动图
- `.webp` — WebP 图片（含动图）
- `.bmp` — BMP 位图

**扫描性能考虑：**
- 首次扫描使用后台线程，显示进度条
- 增量扫描：对比文件修改时间，仅更新变更部分
- 大型仓库（万张以上）：分批扫描，避免 ANR

#### A3 — 仓库元数据管理

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 管理仓库的元信息：路径、名称、添加时间、扫描状态 |
| 存储 | SQLite / Room 数据库 |
| 操作 | 增删改查仓库配置 |

### 模块B：相册管理

#### B1 — 自动相册创建

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 将根仓库中的每个子目录自动映射为一个子相册 |
| 规则 | 子目录名 = 相册名；子目录下所有图片 = 相册内容 |
| 层级 | 采用多层级展示（树形结构），子目录嵌套时形成父子相册关系；若后续效果不佳再考虑平铺 |

**多层级映射示例：**
```
根仓库: /storage/emulated/0/Pictures/
├── 旅行/                    → 相册"旅行"（一级）
│   ├── photo1.jpg
│   ├── 日本/                → 子相册"旅行/日本"（二级）
│   │   ├── tokyo.jpg
│   │   └── osaka.png
│   └── 欧洲/                → 子相册"旅行/欧洲"（二级）
│       └── paris.jpg
├── 美食/                    → 相册"美食"（一级）
│   └── food.gif
└── 壁纸/                    → 相册"壁纸"（一级）
    ├── nature/
    │   └── ...              → 子相册"壁纸/nature"（二级）
    └── abstract/
        └── ...              → 子相册"壁纸/abstract"（二级）
```

**UI 展示：** 相册列表中以缩进或树形展开方式体现多层级关系

#### B2 — 相册缩略图

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 为每个子相册自动设置封面缩略图 |
| 选取策略 | 默认取该目录下第一张图片（按文件名升序排列后的第一张）；后续允许用户自定义封面 |
| 自动更新 | 第一张图片被删除后，自动选取下一张作为封面；**若用户已自定义封面则保持不变** |
| 缓存 | 缩略图需缓存到应用私有目录，避免重复解码 |

#### B3 — 相册分类

| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 用户创建分类标签，将相册归入不同分类以便管理 |
| 数据模型 | 分类（id, name, color/icon）+ 相册-分类关联表 |
| 交互 | 分类以标签页或侧边筛选方式展示 |
| 约束 | 一个相册可属于多个分类（多对多关系） |

#### B4 — 相册排序配置

| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 支持全局默认排序 + 单相册覆盖排序 |

**排序选项：**

| 排序方式 | 说明 | 排序依据 |
|---------|------|---------|
| 按修改时间（旧→新）⭐ | 最早修改的在前（**默认**） | 文件 lastModified |
| 按修改时间（新→旧） | 最近修改的在前 | 文件 lastModified |
| 按文件名（A→Z） | 字母升序 | 文件名（忽略扩展名） |
| 按文件名（Z→A） | 字母降序 | 文件名（忽略扩展名） |
| 按文件大小（小→大） | 从小到大 | 文件字节数 |
| 按文件大小（大→小） | 从大到小 | 文件字节数 |

**配置优先级：** 单相册设置 > 全局设置 > 默认（修改时间升序，旧→新）

### 模块C：图片浏览

#### C1 — 相册网格/列表视图

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 展示相册内的图片缩略图网格，点击进入全屏浏览 |
| 布局模式 | 可切换：每行 2 列（默认）/ 单列列表 |
| 性能 | 缩略图懒加载，RecyclerView 复用，大规模图片不卡顿 |

#### C2 — 全屏浏览

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 点击图片进入全屏模式，隐藏状态栏和导航栏 |
| 适配策略 | 默认 FIT_CENTER（等比缩放完整显示），支持**双指缩放**自由调节（1×~5×），双击切换 FIT↔CROP |
| 缩放复位 | **切换图片时缩放状态重置**为 FIT_CENTER，避免用户困惑 |
| 切换动画 | **无过渡动画**，直接切换，满足快速连续浏览需求 |
| 预加载策略 | 向后预加载 ≥5 页，向前预加载 1-2 页（用户大概率向后翻） |
| 横竖屏 | 跟随传感器自动旋转；UI 布局随横竖屏自适应调整 |

**全屏 UI 布局（竖屏）：**
```
┌─────────────────────┐
│                     │
│                     │
│      图片区域        │  ← 占据绝大部分屏幕
│    （支持双指缩放）   │
│                     │
│                     │
├─────────────────────┤
│ ■━━━━━━○━━━━━━━━━  │  ← 底部简洁进度条（可拖动）
│      12 / 5000      │
└─────────────────────┘
```

**全屏 UI 布局（横屏）：**
```
┌──────────────────────────────────────────┐
│                                          │
│              图片区域                      │
│            （支持双指缩放）                 │
│                                          │
├──────────────────────────────────────────┤
│ ■━━━━━━━━━━○━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│              12 / 5000                    │
└──────────────────────────────────────────┘
```

#### C3 — 图片切换

| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 在全屏模式下提供多种切换图片方式 |

| 切换方式 | 操作 | 备注 |
|---------|------|------|
| 左右滑动 | 手指左右滑动 | **无过渡动画**，直接切换 |
| 点击屏幕 | 点击左侧 1/3→上一张，右侧 2/3→下一张 | 仅在 UI 可见时响应 |
| 音量键 | 音量+ 翻到上一张，音量- 翻到下一张 | 全屏时拦截系统音量 |
| 鼠标滚轮 | 滚轮向下→下一张，向上→上一张 | 外接鼠标（OTG/蓝牙） |

**手势冲突处理方案：**
- 单击：显示/隐藏 UI（进度条、按钮）
- 双击：缩放切换（fit ↔ crop），切换后缩放状态重置
- 双指捏合：自由缩放（pinch-to-zoom）
- 长按：弹出操作菜单（分享、详细信息等）
- 左右滑动：切换图片（无动画，瞬时切换）
- 点击区域翻页仅在 UI 已显示时生效，且仅响应非滑动点击

#### C4 — 自动播放（幻灯片）

| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 按固定时间间隔自动切换图片 |
| 触发 | 全屏模式下点击播放按钮或从菜单启动 |
| 间隔选项 | 1秒 / 2秒 / 3秒 / 5秒 / 10秒 / 自定义 |
| 过渡动画 | **无过渡动画**，直接切换 |
| 循环 | **始终循环**，到达最后一张后自动回到第一张继续播放 |

#### C5 — 进度条导航

| 属性 | 说明 |
|------|------|
| 优先级 | P1 |
| 描述 | 全屏底部显示简洁进度条 |
| 设计原则 | 极简设计，精度要求不高，不干扰图片浏览 |
| 交互 | 显示当前位置数字（如 "12 / 5000"）；拖动滑块快速跳转 |
| 视觉 | 细线进度条 + 小圆形滑块，半透明背景，自动隐藏时不占视觉空间 |
| 约束 | **不支持**拖动预览缩略图，保持简洁 |

### 模块D：设置与配置

#### D1 — 全局设置

| 设置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 默认排序方式 | 枚举 | 修改时间（升序，旧→新） | 新相册的默认排序 |
| 每页相册数量 | 整数 | 20 | 相册列表分页大小 |
| 自动播放间隔 | 整数 | 3 秒 | 幻灯片默认间隔 |
| 音量键翻页 | 开关 | 开启 | 是否使用音量键翻页 |
| 主题模式 | 枚举 | 跟随系统 | 浅色 / 深色 / 跟随系统 |

#### D2 — 单相册设置

| 设置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 排序方式 | 枚举 | 使用全局设置 | 覆盖该相册的排序方式 |
| 自定义封面 | 图片路径 | 自动选择 | 手动指定封面图 |
| 所属分类 | 多选 | 无 | 归入已有分类 |

#### D3 — 相册列表分页设置

| 属性 | 说明 |
|------|------|
| 每页数量 | 可配置，默认 20 个相册/页 |
| 翻页控件 | 底部显示页码导航：`[<] 1 2 3 ... 10 [>]` |
| 跳页 | 点击页码区域弹出输入框，输入目标页数跳转 |
| 持久化 | **不记住**页码，每次进入相册列表默认显示第 1 页 |

### 模块E：界面风格

#### E1 — 设计语言

| 要求 | 实现方式 |
|------|---------|
| 简洁化 | Material Design 3 风格，大面积留白，减少装饰元素 |
| 圆角 | 按钮圆角 12dp，卡片/相册缩略图圆角 8dp，对话框圆角 16dp |
| 配色 | 以图片内容为主，UI 使用中性色（白/灰/黑），主题色点缀 |
| 深色主题 | 支持浅色/深色/跟随系统三种模式；深色模式下**自动检测屏幕类型**：OLED 设备纯黑 `#000000`，LCD 设备深灰 `#121212` |

**色彩方案：**

| 模式 | 背景色 | 卡片色 | 文字色 | 强调色 |
|------|--------|--------|--------|--------|
| 浅色 | #FAFAFA | #FFFFFF | #1A1A1A | #1976D2 |
| 深色 | #121212 | #1E1E1E | #E0E0E0 | #64B5F6 |

#### E2 — 布局模式切换

| 模式 | 描述 |
|------|------|
| 双列网格 | 每行显示两个相册封面，适合快速浏览 |
| 单列列表 | 每行一个相册，左侧缩略图 + 右侧相册名和图片数，信息更丰富 |

**切换方式：** 顶部工具栏右侧切换按钮（网格图标 / 列表图标）

#### E3 — 全屏沉浸

| 属性 | 说明 |
|------|------|
| 状态栏 | 隐藏（SYSTEM_UI_FLAG_FULLSCREEN / WindowInsetsController） |
| 导航栏 | 隐藏（SYSTEM_UI_FLAG_HIDE_NAVIGATION / 手势指示条） |
| 背景色 | 纯黑（#000000），减少 OLED 功耗并突出图片 |

### 模块F：扩展功能（P3 远期）

#### F1 — 远程仓库

| 属性 | 说明 |
|------|------|
| 优先级 | P3（远期） |
| 描述 | 添加局域网内的远程目录作为图片仓库 |
| 协议方案 | ✅ **已确认**：SMB（电脑共享）+ HTTP Server + mDNS（设备间自动发现），详见 `04_远程仓库协议选型建议.md` |
| 缓存 | 浏览过的图片缓存到本地（原图+缩略图），LRU 淘汰策略（默认原图 500MB，缩略图 200MB） |
| 离线策略 | **无网络时显示已缓存的低清版本**，网络恢复后自动切换高清 |
| 体验 | 与本地仓库操作一致，加载时有网络延迟提示 |
| 场景 | 电脑共享文件夹、手机↔手机、手机↔平板 |

#### F2 — 视频播放

| 属性 | 说明 |
|------|------|
| 优先级 | P3（远期） |
| 描述 | 在相册中识别视频文件并播放 |
| 格式优先级 | ① MP4（最优先）→ ② MKV → ③ TS（其它格式暂不支持） |
| 播放器 | Media3 ExoPlayer |
| 交互 | 点击视频进入播放器（类似全屏图片），支持播放/暂停/拖动进度条/音量控制 |

#### F3 — 多用户与权限管理

| 属性 | 说明 |
|------|------|
| 优先级 | P3（远期） |
| 描述 | 支持设备上多个用户账号，不同用户拥有不同的仓库访问权限 |
| 用户模型 | 本地用户账号体系（非联网账号），存储在本地 Room 数据库 |
| 角色划分 | 管理员（admin）和普通用户（viewer）两种角色 |

**角色权限矩阵：**

| 权限 | 管理员 | 普通用户 |
|------|--------|---------|
| 添加/删除仓库 | ✅ | ❌ |
| 浏览所有仓库 | ✅ | 仅授权仓库 |
| 修改全局设置 | ✅ | ❌ |
| 修改单相册设置 | ✅ | ❌ |
| 创建/管理分类 | ✅ | ❌ |
| 浏览图片 | ✅ | 仅授权仓库 |
| 全屏浏览 | ✅ | 仅授权仓库 |
| 自动播放 | ✅ | 仅授权仓库 |
| 添加远程仓库 | ✅ | ❌ |

**关键数据模型：**
- `User（id, name, password_hash, role, created_at）`
- `UserRepositoryAccess（userId, repositoryId）` — 用户可访问的仓库列表

**交互流程：**
1. App 启动 → 若有多用户则显示登录选择界面
2. 仅管理员可进入"用户管理"页面
3. 管理员可创建/删除用户、为用户分配可访问的仓库
4. 普通用户登录后仅能看到被授权的仓库

## 三、模块依赖关系

```
模块A（仓库管理）
    │
    ▼
模块B（相册管理） ────── 模块D（设置配置）
    │                        │
    ▼                        ▼
模块C（图片浏览） ◄──────────┘
    │
    ▼
模块E（界面风格） ← 全局影响所有模块
    │
    ▼
模块F（扩展功能） ← 远期，依赖模块 A/B/C
    ├── F1 远程仓库 ← 依赖 A（仓库管理抽象）
    ├── F2 视频播放 ← 依赖 C（全屏浏览扩展）
    └── F3 多用户权限 ← 依赖 A/D（仓库+设置权限隔离）
```

**开发顺序建议：** A → B → C → D → E → F

## 四、技术栈建议（Android）

| 层级 | 技术选型 |
|------|---------|
| 语言 | Kotlin |
| UI 框架 | Jetpack Compose（优先）/ 传统 XML + ViewBinding |
| 图片加载 | Coil（Compose）/ Glide（View） |
| 数据库 | Room（SQLite） |
| 异步 | Kotlin Coroutines + Flow |
| 权限 | Accompanist Permissions / ActivityResult API |
| 视频播放 | Media3 ExoPlayer（远期） |
| 网络访问 | OkHttp + Retrofit / JCIFS（SMB 远期） |
| 最低 SDK | API 29（Android 10） |
| 目标 SDK | API 35（Android 15） |
| 深色主题 | Material 3 Dark Theme + 跟随系统 |
