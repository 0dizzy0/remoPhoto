## 三、关键技术方案

### 3.1 文件扫描策略

```
┌─────────────────────────────────────────────┐
│              FileScanner                     │
│                                              │
│  1. 读取根仓库路径列表（Room）                 │
│  2. 对每个仓库路径：                          │
│     a. 递归遍历子目录                         │
│     b. 过滤图片文件（按扩展名 + MIME 类型）     │
│     c. 对比数据库已有记录（lastModified）     │
│        ├── 新增文件 → INSERT                 │
│        ├── 修改文件 → UPDATE                 │
│        └── 已删除   → DELETE                 │
│  3. 更新仓库 lastScanTime 和 imageCount      │
│                                              │
│  运行策略：                                   │
│  • 首次启动 → 全量扫描（显示进度）              │
│  • 后续启动 → 增量扫描（对比时间戳）            │
│  • 用户手动 → 全量重新扫描                     │
│  • 扫描在后台线程（Dispatchers.IO + 协程）     │
└─────────────────────────────────────────────┘
```

**图片格式过滤（已确认）：**
```kotlin
val SUPPORTED_EXTENSIONS = setOf(
    "png", "jpg", "jpeg", "gif", "webp", "bmp"
)
```

**性能优化：**
- 使用 `Files.walkFileTree()` 或 `File.walkTopDown()` 高效遍历
- 批量数据库操作（一次 Transaction 内完成，每 500 条 commit 一次）
- 扫描过程中不加载图片到内存，仅记录元数据
- 支持 10,000+ 相册、每个相册 10,000+ 张图片的量级
- 大型仓库分批扫描，WorkManager 后台任务，前台显示进度通知
- **封面自动更新**：检测到封面图片文件被删除时 → 自动选取目录下一张图片 → 更新 AlbumEntity.coverImagePath；若封面为用户自定义则跳过

**GIF / WebP 动图支持：** Coil 自带 GIF 支持（`coil-gif`），WebP 动图由框架原生支持，自动检测并播放。

**性能考虑（万级图片量）：**

- `beyondBoundsPageCount` 设置为 ≥5，确保向后翻页无等待

// 远程图片加载时优先本地缓存，网络不可用时 fallback 到低清版本
fun loadRemoteImage(url: String, localCachePath: String): ImageRequest {
    val context = this
    return ImageRequest.Builder(context)
        .data(url)
        .diskCacheKey(localCachePath)
        .error(
            // 网络不可用时，尝试加载本地缓存的低清版本
            ImageRequest.Builder(context)
                .data(File(localCachePath))  // 上次缓存的缩略图/低清版本
                .build()
        )
        .build()
}

**双指缩放实现：**
```kotlin
@Composable
fun ZoomableImage(
    image: ImageItem,
    onTap: () -> Unit,
    onDoubleTap: () -> Unit
) {
    var scale by remember { mutableFloatStateOf(1f) }
    var offset by remember { mutableStateOf(Offset.Zero) }
    
    val transformableState = rememberTransformableState { zoomChange, panChange, _ ->
        scale = (scale * zoomChange).coerceIn(1f, 5f)
        offset = if (scale > 1f) offset + panChange else Offset.Zero
    }
    
    Box(
        modifier = Modifier
            .fillMaxSize()
            .transformable(transformableState)
            .pointerInput(Unit) {
                detectTapGestures(
                    onTap = { onTap() },
                    onDoubleTap = { onDoubleTap() }
                )
            }
    ) {
        AsyncImage(
            model = image.filePath,
            contentDescription = image.fileName,
            modifier = Modifier
                .fillMaxSize()
                .graphicsLayer(
                    scaleX = scale,
                    scaleY = scale,
                    translationX = offset.x,
                    translationY = offset.y
                ),
            contentScale = ContentScale.Fit
        )
    }
}
```

**手势处理分层：**
| 层级 | 手势 | 响应 |
|------|------|------|
| Pager 层 | 水平滑动 | 切换图片（无动画，瞬时到位） |
| 图片层 | 双指捏合 | 自由缩放（1× ~ 5×） |
| 图片层 | 双击 | 缩放切换 FIT↔CROP（重置缩放状态） |
| 图片层 | 单击 | 显示/隐藏 UI 覆盖层 |
| 图片层 | 长按 | 弹出操作菜单 |
| 全局 | 音量键 | 翻页（拦截 KeyEvent，音量+→上一张，音量-→下一张） |
| 全局 | 鼠标滚轮 | 翻页（onGenericMotionEvent，滚轮下→下一张，上→上一张） |

**图片切换行为确认：**
- 切换图片时缩放状态**重置**（回到 FIT_CENTER, scale=1）
- 无过渡动画，直接显示下一张（满足快速连续浏览需求）
- 预加载优先级：向后 > 向前

## 五、存储权限适配

### 5.1 统一方案：SAF + 持久化授权（API 29+）

> **决策确认：** 仅支持 Android 10（API 29）及以上，统一使用 SAF，不兼容更低版本。

```
1. 用户点击"添加仓库"
2. 启动 SAF 目录选择器 (ACTION_OPEN_DOCUMENT_TREE)
3. 用户选择目录并授权
4. 通过 contentResolver.takePersistableUriPermission() 持久化授权
5. 保存 Uri 到数据库
6. 通过 DocumentFile API 访问文件树
7. 对于需要绝对路径的场景，通过 contentResolver.query() 获取实际路径
```

**优点：** 无需 `MANAGE_EXTERNAL_STORAGE` 权限，Google Play 审核友好，用户可控

**注意事项：**
- SAF 大目录遍历性能可能低于直接文件访问，万级图片量需异步处理
- 部分厂商 ROM（如 MIUI、ColorOS）的 SAF 实现可能有差异，需测试
- 用户卸载重装后持久化授权可能失效，需引导重新授权

    // LCD 设备（部分中低端机型）返回 false
    val displayManager = getSystemService(Context.DISPLAY_SERVICE) as DisplayManager
    // 简化判断：API 31+ 可通过 Display.getHdrCapabilities() 辅助判断
    // 默认假定为 OLED（市面上主流设备已是 OLED）
    return true  // Safe default for most modern devices
}

// 主题切换（支持浅色/深色/跟随系统，OLED 自适应）
@Composable
fun RemoPhotoTheme(
    themeMode: ThemeMode,  // LIGHT, DARK, SYSTEM
    content: @Composable () -> Unit
) {
    val context = LocalContext.current
    val darkTheme = when (themeMode) {
        ThemeMode.LIGHT -> false
        ThemeMode.DARK -> true
        ThemeMode.SYSTEM -> isSystemInDarkTheme()
    }
    // 深色模式下根据屏幕类型选择背景色
    val colors = if (darkTheme) {
        if (context.isOledDisplay()) DarkColors  // 纯黑 #000000
        else DarkColors.copy(background = Color(0xFF121212))  // 深灰 LCD
    } else {
        LightColors
    }
    MaterialTheme(
        colorScheme = colors,
        shapes = Shapes,
        content = content
    )
}

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| SAF 大目录遍历性能 | 万级图片扫描慢 | 异步 WorkManager 后台扫描 + 分批 commit |
| SAF 厂商 ROM 兼容 | 不同系统 SAF 实现差异 | 小米/OPPO/vivo 等主流 ROM 专项测试 |
| GIF/WebP 动图性能 | 同时播放多动图卡顿 | 仅全屏时播放动图，列表仅显示静态缩略图 |
| 万级图片内存管理 | OOM 风险 | Coil 内存缓存限制 128MB + 弱引用 + bitmap reuse |
| 多层级相册 UI | 树形结构交互复杂 | 先用简单缩进方案，后续迭代优化 |
| 音量键拦截失败 | 部分 ROM 无法拦截 | 设置中提供开关；ROM 不支持时 fallback 为媒体键 |
| SAF 授权失效 | 卸载重装后 Uri 不可用 | 检测授权有效性，失效时引导用户重新授权 |
| 无过渡动画的感知 | 用户可能觉得生硬 | 保持极简理念，通过 Pager snap 确保瞬间到位 |
| 多用户数据安全 | 密码泄露、权限绕过 | 密码 SHA-256+Salt 哈希存储；仓库访问入口做权限校验 |
| OLED 检测误判 | LCD 设备误用纯黑背景 | 默认假定 OLED（主流设备），同时提供手动设置选项 |
| 远程离线降级 | 缓存未命中时无图可显示 | 首次浏览时主动预缓存缩略图；显示"无网络，暂无缓存"提示 |
| mDNS 兼容性 | 部分路由器屏蔽 mDNS | 同时提供手动输入 IP:端口 的备选连接方式 |
