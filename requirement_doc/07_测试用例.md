# remoPhoto 测试用例

更新时间：2026-06-23

## 1. 测试目标

首个 alpha 的测试目标是证明：

- 安装包可构建、可签名、可安装。
- 本地仓库、相册、全屏浏览和设置页核心流程可用。
- 远程仓库在可信局域网内可发现、连接、分页、缓存和离线恢复。
- 数据库导入导出不会破坏现有数据。
- 关键问题能通过日志定位。

当前自动化覆盖较少，alpha 前以手工 smoke test + 少量 JVM 单测为主，alpha 后逐步补自动化。

## 2. 测试分层

| 层级 | 工具 | 当前状态 | 目标 |
| --- | --- | --- | --- |
| JVM 单元测试 | JUnit | 已有少量测试 | 分页、排序、查找、DTO 兼容、缓存策略 |
| Room 迁移测试 | AndroidX Test + MigrationTestHelper | 暂无 | 覆盖 v1->v4 迁移 |
| Compose UI 测试 | Compose UI Test | 暂无 | 覆盖设置、相册、分页、查找 |
| HTTP 契约测试 | MockWebServer 或本地 server | 暂无 | 覆盖新旧 API 和错误响应 |
| 真机 smoke test | adb + 手工操作 | 已有历史验证 | alpha 发布前必须执行 |

## 3. 发布前本地门禁

```powershell
.\gradlew.bat :app:testDebugUnitTest :app:lintDebug :app:assembleRelease --console=plain
```

验收：

- `testDebugUnitTest` 通过。
- `lintDebug` 无 error。
- `assembleRelease` 通过。
- 如果已配置签名，应生成 signed APK/AAB。

## 4. 通用 adb 模板

```powershell
$serial = "<adb-serial>"
$apk = "app\build\outputs\apk\release\<signed-apk-name>.apk"

adb -s $serial install -r $apk
adb -s $serial logcat -c
adb -s $serial shell am force-stop com.remophoto
adb -s $serial shell am start -W -n com.remophoto/.MainActivity
adb -s $serial shell uiautomator dump /sdcard/remophoto-ui.xml
adb -s $serial logcat -b crash -d
adb -s $serial shell dumpsys meminfo com.remophoto
```

每次测试至少记录：

- 测试日期。
- commit hash。
- APK 版本号。
- 设备型号和 Android 版本。
- 本地/远程仓库规模。
- crash buffer 是否新增异常。
- 关键 AppLogger 摘要。

## 5. Alpha Smoke Test

### TC-ENV-001 冷启动和覆盖安装

步骤：

1. 覆盖安装 release APK。
2. 清空 crash buffer。
3. 冷启动应用。
4. 进入首页、设置页、仓库管理页。

断言：

- 安装成功。
- 冷启动无崩溃。
- 页面可切换。
- crash buffer 无新增 `FATAL EXCEPTION`。

### TC-LOCAL-001 添加本地仓库并扫描

步骤：

1. 添加一个包含图片和子目录的本地仓库。
2. 触发扫描。
3. 观察仓库卡片进度和完成状态。
4. 进入相册列表。

断言：

- SAF 授权成功。
- 扫描任务进入前台任务。
- 相册数量和图片数量合理。
- 日志包含 repoId、阶段、耗时和数量。

### TC-LOCAL-002 大仓库扫描稳定性

步骤：

1. 使用至少数千张图片的仓库。
2. 启动扫描。
3. 切到后台、息屏，再回到应用。
4. 观察进度和最终结果。

断言：

- 无 ANR/OOM。
- 应用进程保持可用。
- 扫描失败时旧索引仍可用。
- `dumpsys meminfo` 记录内存快照。

### TC-ALBUM-001 相册分页和查找

步骤：

1. 进入包含多页相册的仓库。
2. 切换第 1 页、第 2 页和末页。
3. 输入相册名称片段执行查找。

断言：

- 总页数正确。
- 翻页后首项发生变化。
- 查找只跳转页码，不过滤列表。
- 无命中时显示提示。

### TC-ALBUM-002 相册排序

步骤：

1. 修改相册列表排序为名称、图片数量、修改时间。
2. 分别测试升序和降序。
3. 进入相册后检查图片排序不被相册排序影响。

断言：

- 相册排序持久化。
- 图片排序独立。
- 同值排序稳定。

### TC-VIEWER-001 全屏浏览

步骤：

1. 进入任意相册。
2. 打开第一张图片。
3. 测试滑动、点击区域、音量键、鼠标滚轮翻页。
4. 测试双指缩放和双击切换。
5. 测试自动播放开始/暂停。

断言：

- 翻页方向符合设置。
- 缩放状态下不误触发翻页。
- 切图时没有明显状态栏跳动。
- 无崩溃。

### TC-SET-001 设置页和缓存统计

步骤：

1. 进入设置页。
2. 修改主题、排序、分页数量、播放间隔。
3. 查看存储空间和远程缓存栏目。
4. 从底部导航切回相册列表。

断言：

- 设置持久化。
- 缓存统计不长期停留在“计算中”。
- 设置页切回相册列表成功。

### TC-REMOTE-001 远程服务启动和发现

前置：

- 两台 Android 设备处于同一可信局域网。

步骤：

1. 设备 A 开启远程服务。
2. 设备 B 打开添加远程仓库。
3. 等待 mDNS 发现。
4. 或手动输入 A 的 IP 和端口。

断言：

- A 的远程服务启动成功。
- B 可以发现或手动连接 A。
- B 不会把自身加入为远程仓库。
- 日志包含 mDNS 注册/发现信息。

### TC-REMOTE-002 远程相册分页和缓存

步骤：

1. 进入远程仓库。
2. 等待相册列表加载。
3. 切换第 1 页、第 2 页和末页。
4. 退出再进入远程仓库。

断言：

- 首次同步成功。
- 分页栏显示真实总页数。
- 二次进入优先显示缓存。
- 缓存命中时不重复触发完整同步。

### TC-REMOTE-003 远程离线和恢复

步骤：

1. 在分类中加入一个远程相册。
2. 关闭网络或停止远程服务。
3. 进入该分类。
4. 恢复网络或远程服务。
5. 重新同步。

断言：

- 离线时分类中不显示远程相册。
- 缓存数据不被清空。
- 恢复后远程仓库状态回到可连接。

### TC-BACKUP-001 数据库导出

步骤：

1. 进入设置页。
2. 导出数据库。
3. 检查导出文件存在。

断言：

- 导出成功。
- manifest 存在。
- 日志包含导出开始、数据库版本、完成状态。
- 不导出远程凭据。

### TC-BACKUP-002 数据库导入校验

步骤：

1. 选择一个有效备份文件导入。
2. 冷启动应用。
3. 检查仓库、相册、分类和设置。

断言：

- 导入成功或失败提示明确。
- 失败时旧数据仍可用。
- 远程连接状态重置为断开。
