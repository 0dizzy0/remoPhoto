# 远程仓库协议决策记录

更新时间：2026-06-23

## 1. 当前结论

首个 alpha 采用：

- Android 设备之间：HTTP Server + mDNS。
- PC/NAS 共享：SMB 后延到 beta 或后续阶段。
- 公网访问：不支持。
- HTTPS/TLS：不属于 alpha。

此决策优先满足个人局域网内多 Android 设备互相浏览图片的场景。

## 2. 场景拆分

| 场景 | 当前策略 | 说明 |
| --- | --- | --- |
| 手机 -> 手机 | HTTP + mDNS | alpha 主场景 |
| 手机 -> 平板 | HTTP + mDNS | alpha 主场景 |
| 平板 -> 手机 | HTTP + mDNS | alpha 主场景 |
| 手机 -> PC/NAS | SMB | 后延 |
| 公网访问 | 不支持 | 安全和复杂度不适合 alpha |

## 3. 为什么选择 HTTP + mDNS

优势：

- Android 端实现可控，不依赖系统 SMB 能力。
- 服务端和客户端都在应用内，方便添加日志和兼容策略。
- mDNS 可以减少用户手动输入 IP 的成本。
- HTTP 图片流适合接入 Coil 自定义 Fetcher。
- 元数据可缓存到 Room，离线时仍能展示已缓存状态。

代价：

- 当前是明文 HTTP，只适合可信局域网。
- 不适合公网。
- 需要处理 Android 后台、息屏、Wi-Fi、多播锁和前台服务限制。
- 不解决 PC/NAS 共享场景。

## 4. 服务发现

服务类型：

```text
_remophoto._tcp
```

服务 TXT 建议字段：

| 字段 | 说明 |
| --- | --- |
| `deviceName` | 用户配置或设备名称 |
| `instanceId` | 本机实例标识，用于过滤自身 |
| `apiVersion` | API 版本 |
| `port` | HTTP 端口 |

客户端策略：

- 优先使用 mDNS 发现。
- mDNS 无结果时，可允许手动 IP + 端口。
- 发现结果必须过滤自身。
- 发现失败要记录日志，但不阻塞手动添加。

## 5. HTTP API

### `GET /api`

健康检查。

返回建议：

```json
{
  "name": "remoPhoto",
  "apiVersion": 1,
  "deviceName": "Phone"
}
```

### `GET /api/albums`

返回本机本地仓库中的相册元数据，不二次转发远程仓库。

字段：

| 字段 | 说明 | 兼容性 |
| --- | --- | --- |
| `id` | 服务端相册 ID | 必需 |
| `name` | 相册名 | 必需 |
| `path` | 稳定远程路径 | 必需 |
| `parentId` | 服务端父相册 ID | 可空 |
| `imageCount` | 图片数量 | 必需 |
| `coverImageId` | 封面图片 ID | 可空，旧服务端可能没有 |
| `lastModified` | 相册最近修改时间 | 可选，旧服务端缺失时按 0 处理 |

### `GET /api/albums/{id}/images`

返回指定相册图片元数据。

### `GET /api/images/{id}`

返回原图流。

### `GET /api/thumb/{id}`

返回缩略图流。服务端应下采样，避免把大图作为封面完整传输。

## 6. 同步策略

相册同步：

- 拉取远程相册列表。
- 用远程稳定路径复用本地相册主键。
- 只删除服务端已移除的相册。
- 保留未变化相册的分类关联和缓存。
- 批量写入 Room 事务。

图片同步：

- 进入远程相册时按需拉取图片列表。
- 使用服务端图片 ID 映射远程图片 URL。
- 保留已有缓存索引。

缓存策略：

- 远程相册元数据写入 Room。
- 默认缓存有效期 30 分钟。
- 命中缓存时先展示本地数据。
- 过期时先展示旧缓存，再后台刷新。
- 手动刷新强制重新同步。

## 7. 离线行为

- 远程连接失败时标记为 `ERROR` 或 `DISCONNECTED`。
- 远程仓库列表可展示已缓存数据和离线提示。
- 分类页只显示本地仓库和当前已连接远程仓库的相册。
- 远程缓存不应因为单次网络错误被清空。

## 8. 安全边界

Alpha 版本必须明确说明：

- HTTP 明文传输仅适用于可信局域网。
- 不要在公共 Wi-Fi 或公网暴露远程服务。
- 当前没有访问令牌、TLS 或用户权限模型。
- 导出数据库不包含远程凭据。

后续安全增强方向：

- 局域网配对码。
- 访问令牌。
- HTTPS/TLS。
- 只允许白名单设备访问。

## 9. SMB 后延原因

SMB 更适合 PC/NAS 场景，但 alpha 阶段后延，原因：

- Android 端 SMB 依赖和兼容性成本高。
- 认证、权限、编码、路径和大文件行为复杂。
- 当前首要目标是先验证 Android 设备间远程仓库闭环。

SMB 进入 beta 前需要单独评估依赖、许可证、性能和安全边界。

## 10. 验收要点

- 两台 Android 设备在同一局域网内可发现彼此。
- 可手动添加 IP。
- 不能添加自身。
- `/api` 健康检查可用。
- `/api/albums` 不返回远程中继仓库。
- 远程相册可分页、显示封面、进入图片列表。
- 网络断开后不崩溃、不清空缓存。
- 网络恢复后可重新同步。
