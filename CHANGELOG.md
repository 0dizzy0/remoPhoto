# 更新日志

本项目遵循“面向用户的变化进入更新日志，开发过程进入项目状态或归档”的原则。版本格式计划采用语义化版本。

## [Unreleased]

暂无。

## [0.1.0] - 2026-07-08

### 新增

- 增加发布候选 APK 一键 Smoke 脚本，自动收集签名、安装、版本、冷启动、UI dump、截图、Crash buffer、进程日志、应用文件日志、基础 Release 日志隐私扫描和内存快照证据。

### 修复

- 修复 Android 10（API 29）DownloadsProvider 返回 `raw:` document ID 时，仓库名和相册名错误显示完整内部路径的问题。

### 验证

- 增加 5 条 SAF document ID 名称提取 JVM 测试；完整 JVM 测试增至 24 条。
- 完成 API 29 模拟器全新安装、冷启动、SAF 授权、扫描、相册、图片和全屏浏览回归。
- 完成 `0.1.0` 候选一键 Smoke、基础 Release 日志隐私扫描和 Android 集成测试，覆盖 Release APK 基础证据、Room migration、导入失败回滚和有效备份恢复。
- 完成 `0.1.0` 候选主力真机本地 SAF Smoke，覆盖授权、扫描、相册、图片网格、全屏和基础日志隐私检查。
- 完成 `0.1.0` 候选双机远程 Smoke，覆盖 HTTP 前台服务、mDNS 自动发现、远程同步、远程图片全屏、服务离线恢复和远程流程日志隐私。
- 补充有效备份恢复 Android 集成测试，覆盖成功导入后的数据库、设置、SQLite 完整性和临时目录清理；vivo Android 12 真机执行 3/3 通过。
- 完成 `0.1.0-beta.1 -> 0.1.0` 同签名覆盖升级数据保留验证，本地仓库、远程仓库、图片数量和连接状态保留。

## [0.1.0-beta.1] - 2026-07-06

### 变更

- 开发版本更新为 `0.1.0-beta.1`（`versionCode = 5`），进入范围冻结和发布候选验证阶段。
- 备份导入流程增加可观测阶段回调，用于在隔离环境中重复验证数据库与设置替换失败后的完整回滚。

### 验证

- 增加备份导入失败回滚 Android 集成测试，验证原数据库、原设置、SQLite 完整性和临时目录清理。
- 完成上一公开 Alpha 覆盖升级、有效备份恢复、主力真机回归和 Android 16/12 双机远程回归。

## [0.1.0-alpha.4] - 2026-07-02

### 新增

- 增加最小 GitHub Actions CI，对 push、Pull Request 和手动触发运行 JVM 测试、Android Lint 与 Debug 构建，并保留失败报告。
- 增加相册分页边界、稳定排序、Unicode/空白名称和模糊定位 JVM 测试。
- 增加备份 manifest/version、条目白名单、路径穿越、重复条目和损坏 ZIP JVM 测试。
- 增加 Room v3→v4 真机 migration 测试，验证相册最近修改时间回填和最终 Room schema。

### 变更

- 开发版本更新为 `0.1.0-alpha.4`（`versionCode = 4`）。
- 相册树排序提取为纯规则，并为同排序值增加确定性次级顺序。
- 备份格式、ZIP 类型、条目白名单和路径安全规则提取为可测试的统一导入策略。
- Debug 包使用 `com.remophoto.debug`，可与正式签名版本并存，避免调试安装破坏公开版本数据。

### 文档

- 增加适合个人小型项目的精简发布路线：保留签名、升级、备份、隐私和关键回归门禁，按变更风险决定双机、兼容性和 UI 专项测试。
- 将默认版本路线收敛为 `alpha.4 → beta.1 → 0.1.0`，仅在出现高风险修复时增加额外 Beta 或 RC。
- 增加 `alpha.4` 候选回归记录和 Release Notes。

## [0.1.0-alpha.3] - 2026-06-29

### 文档

- 将项目文档统一归档到 `docs/`，按产品、架构、项目、测试、发布、设计和归档分类。
- 增加 GitHub 首页、贡献指南、安全说明、Issue 模板和 Pull Request 模板。
- 扩充核心功能、异常路径、日志证据和自动化优先级测试用例。
- 增加 AI 辅助开发透明说明，记录代理、Skills、MCP 状态、工具边界和人工验收责任。

### 修复

- 清理 Android Lint 的 27 条告警，修正 Compose 参数规范、高频状态读取、mDNS 实例标识、WakeLock 超时、冗余 SDK 判断、无用资源和单色图标。
- 增加两项局部 Lint 例外：API 36 升级留作独立兼容任务；自适应图标保留 AAPT 要求的 `v26` 目录。
- 修复 Release 文件日志仍记录图片文件名和相册/仓库实体名的问题，并补充带空格名称的脱敏测试。
- 修复 Release 文件日志仍记录用户自定义分类名称的问题。
- 修复远程全屏原图未纳入设置页专用缓存统计和清理的问题，并让清理后的统计排除 Coil 索引元数据。
- 修复关闭远程服务后 started Service 和自愈回调残留的问题。

### 发布准备

- 首个 Alpha 版本更新为 `0.1.0-alpha.3`（`versionCode = 3`）。
- 增加 MIT License。
- 配置并归档 Room v4 schema，构建不再报告缺少 `schemaLocation`。
- 增加本地 Release keystore 初始化脚本和环境变量签名入口，成功生成并校验 signed APK。
- Release 构建关闭 Logcat 和 DEBUG 文件日志，并统一脱敏 URI、路径、IP 和用户自定义名称。
- 禁止系统云备份及设备迁移提取应用数据；保留明文 HTTP，但明确仅限可信局域网。
- 完成仓库隐私基线审计，强化环境变量、数据库和签名属性忽略规则，并增加不回显匹配内容的审计脚本。
- 局域网发现默认改用稳定随机别名，不再广播设备型号；设置页明确提示远程共享范围。
- 图片分享不再回退到暴露文件路径的 `file://` URI。
- 数据库导出前明确提示备份包含的敏感元数据和未加密属性。
- 首个 Alpha 的 Lint 门禁、Android 12 真机核心回归、双机远程回归和发现问题修复复测已完成。
