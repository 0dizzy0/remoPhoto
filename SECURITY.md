# 安全说明

更新时间：2026-07-15

## 当前支持状态

remoPhoto 仍处于 Alpha 准备阶段，暂不承诺稳定的安全更新周期。发现安全问题时，请优先使用 GitHub Private Vulnerability Reporting；仓库启用该能力前，不要在公开 Issue 中披露可被直接利用的细节、凭据或个人数据。

报告中建议包含：

- 受影响版本或 commit。
- Android 版本和设备环境。
- 可复现步骤及影响范围。
- 已脱敏的日志或最小复现样本。
- 可能的缓解方式。

## 已知安全边界

- remoPhoto 设备间远程仓库使用明文 HTTP，没有 TLS、访问令牌或用户权限模型。
- PC/NAS 远程仓库只支持 SMB2/3 普通账号和只读访问，不支持 SMB1、Guest/匿名、Kerberos、DFS 或远端写入。是否要求 SMB 消息签名取决于服务端策略，当前客户端不额外强制。
- 远程服务仅设计用于可信局域网，不应暴露到公网或公共 Wi-Fi。
- 应用会索引用户授权目录中的文件名、目录结构和图片元数据，这些信息应视为敏感数据。
- 数据库备份包含仓库、相册、分类、SMB 主机/共享/用户名等连接元数据和设置；HTTP/SMB 远程凭据不进入备份。SMB 密码只保存在 Android Keystore。
- 系统云备份和设备迁移已禁用应用数据提取；跨设备迁移应使用应用内导入导出。
- 明文流量因任意局域网地址发现机制而保留，无法用静态域名白名单表达，后续版本应评估认证和加密。

## 日志与附件

提交问题前必须移除：

- 完整文件路径、目录名和相册名。
- 设备 IP、主机名和端口组合。
- 数据库文件、备份文件和签名材料。
- 能识别个人身份的图片或元数据。

必要时可保留稳定 ID、数量、耗时、错误类型和经过散列或截断的路径摘要。

Release 构建默认不输出 Logcat 和 DEBUG 文件日志，并对 URI、路径、IP、主机及用户自定义名称做统一脱敏。该保护不能替代提交附件前的人工检查。

## 仓库隐私审计

提交或公开仓库前执行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\audit-repository-privacy.ps1
```

脚本检查已跟踪及未忽略待提交文件中的私钥、常见令牌、硬编码凭据、邮箱、电话号码、本机用户目录和高风险文件类型，只输出类别、文件与行号，不输出命中内容。示例私有网段地址会作为提示保留，不视为个人地址或发布阻塞项。
